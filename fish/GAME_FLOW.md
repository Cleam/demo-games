# GAME_FLOW.md

## 1. 文档定位

本项目是一个竖屏 H5 海底进化战斗试玩展示项目，包含两条固定流程：

* `?mode=win`：进化 → 鱼王 Boss 战 → 胜利奖励 → CTA。
* `?mode=lose`：进化 → 最终对抗 → 挑战失败 → 重试或退出。

本文档是**交互、流程、状态与验收的唯一事实来源**。

资料优先级：

```text
1. GAME_FLOW.md       // 交互、状态、流程、功能验收
2. VIDEO_ANALYSIS.md  // 画面、节奏、镜头、UI 参考
3. assets/v1.mp4      // 胜利流程视觉参考
4. assets/v2.mp4      // 失败流程视觉参考
5. assets/images      // 实际可用美术资源
```

视频仅用于视觉还原，不应被直接播放、嵌入或伪装成交互试玩内容。

---

## 2. 项目目标与边界

### 2.1 项目目标

实现一个可在手机浏览器运行的竖屏 H5 试玩页面：

```text
初始战斗
→ 单位逐步解锁 / 进化
→ 最终战斗
→ 胜利或失败结算
→ CTA 或重新挑战
```

重点还原：

* 海底战斗场景；
* 鱼类角色帧动画；
* 攻击、命中、受击、死亡、必杀特效；
* 底部生物进化卡片；
* Boss 血条、体力条、吞噬时间；
* 成功奖励弹窗、失败弹窗、CTA 页面；
* 参考视频中的大致节奏与画面层级。

### 2.2 不在本期实现范围

不需要实现：

* 真实养成数值；
* 随机掉落；
* 服务端接口；
* 存档；
* 账号系统；
* 真实关卡系统；
* 复杂 AI；
* 物理碰撞；
* 自由移动；
* 多人互动；
* 完整广告平台 SDK。

所有战斗结果均由配置化时间轴（Timeline）控制。

## 3. 资源使用规则

## 3.1 原始资源目录

```text
assets/
├── v1.mp4
├── v2.mp4
└── images/
    ├── bg.png
    ├── sk.png
    ├── fish_11002/
    ├── fish_11003/
    ├── fish_14104/
    ├── fish_15101/
    ├── fish_15102/
    └── skin_fish_151011/
```

角色资源主要为 PNG 帧序列，包含：

```text
hero/
├── idle
├── move
├── atk
├── ult
├── ultb
└── die

effect/
├── eff_atk
├── eff_hit
└── eff_ult
```

不同 fish ID 的实际动作和帧数可能不同，必须以文件扫描结果为准。

## 3.2 资源加载要求

* 原始 `assets/images` 不允许修改、删除、重命名或覆盖。
* 必须忽略 `Thumbs.db`。
* 不允许手写大量逐帧资源路径。
* 应通过脚本扫描资源目录，生成资源清单（Manifest）。
* 帧序列按自然顺序加载，例如：

```text
fish_11002-idle_00.png
fish_11002-idle_01.png
fish_11002-idle_02.png
...
```

* 正式试玩不得一次性加载所有鱼类的全部帧序列。
* 应优先加载当前流程需要的角色和特效，后续角色可提前预加载或按需加载。
* 角色、特效、UI 的映射关系必须放在配置文件中，不应散落在业务代码中。

## 3.3 角色槽位与资源映射

流程使用角色槽位，不直接在流程代码中硬编码 fish ID。

```text
starter        初始单位
evolution_1    第 1 次进化单位
evolution_2    第 2 次进化单位
evolution_3    第 3 次进化单位
final_actor    最终进化单位
boss_win       胜利流程中的鱼王 Boss
enemy_lose     失败流程中的灰色巨鳄 / 最终敌人
```

可用主角色资源池：

```text
fish_11002
fish_11003
fish_14104
fish_15101
fish_15102
skin_fish_151011
```

映射原则：

* `fish_11002`、`fish_11003`、`fish_14104`、`fish_15101`、`fish_15102` 优先用于主流程角色。
* `skin_fish_151011` 默认视为特殊皮肤或 Boss / 敌方候选资源。
* 最终具体映射以资源预览后的 `assetMapping` 配置为准。
* 同一个角色在同一时刻不得重复作为不同阵营的单位出现，除非视觉参考明确要求。
* 若某角色缺少指定动作，按“动作降级规则”处理。

## 3.4 动作降级规则

| 目标动作        | 优先资源            | 缺失时降级方案                |
| ----------- | --------------- | ---------------------- |
| 待机 `idle`   | `hero/*-idle_*` | 使用首帧 + 轻微上下浮动          |
| 移动 `move`   | `hero/*-move_*` | 使用 `idle` + Tween 位移   |
| 普攻 `attack` | `hero/*-atk_*`  | 使用 `ult` 或 `idle` + 前冲 |
| 必杀 `ult`    | `hero/*-ult_*`  | 使用 `atk` + 强化特效        |
| 受击 `hit`    | `eff_hit/*`     | 白闪、抖动、后退               |
| 死亡 `die`    | `hero/*-die_*`  | 缩小、淡出、下沉               |
| 攻击特效        | `eff_atk/*`     | 简单水波 / 光圈              |
| 必杀特效        | `eff_ult/*`     | 放大攻击特效 + 屏幕闪光          |

## 4. 运行模式与入口

## 4.1 URL 参数

```text
?mode=win
?mode=lose
```

规则：

* 未传 `mode` 时默认 `win`。
* 未知 mode 自动降级为 `win`。
* 同一份工程必须支持两种模式。
* 两种模式必须复用角色系统、时间轴系统、UI 系统、战斗系统和清理逻辑。

## 4.2 音频启动规则

参考视频未展示明确开始按钮。

因此：

* 游戏流程默认自动开始。
* 首次用户触摸页面后解锁音频播放权限。
* 首次触摸不得重置、暂停或跳过战斗流程。
* 没有用户触摸时，允许无声播放流程。

## 6. 全局状态机

```text
BOOT
→ LOADING
→ PLAYING
→ FINAL_BATTLE
→ RESULT_WIN / RESULT_LOSE
→ CTA

RESULT_LOSE
→ RESTARTING
→ PLAYING
```

状态说明：

| 状态             | 职责              |
| -------------- | --------------- |
| `BOOT`         | 读取 URL 参数、初始化配置 |
| `LOADING`      | 加载当前 mode 必需资源  |
| `PLAYING`      | 普通鱼群战斗、进化解锁     |
| `FINAL_BATTLE` | Boss 战或最终敌人对抗   |
| `RESULT_WIN`   | 胜利奖励弹窗          |
| `RESULT_LOSE`  | 失败弹窗            |
| `CTA`          | 最终广告引导页         |
| `RESTARTING`   | 清理旧状态并重新播放失败流程  |

状态切换必须由统一控制器管理。

禁止由 UI 按钮、角色对象或特效对象私自修改全局状态。

## 7. 通用交互规则

## 7.1 自动播放内容

以下行为均由时间轴自动触发：

* 初始角色出现；
* 普通敌鱼出现；
* 角色攻击；
* 敌方受击和退出；
* 进化；
* 进化卡解锁；
* Boss / 最终敌人出现；
* 血条、体力条、吞噬时间变化；
* 胜利或失败结算；
* 弹窗展示。

## 7.2 用户可点击内容

| 位置         | 行为                    |
| ---------- | --------------------- |
| 首次触摸屏幕     | 解锁音频播放，不改变流程          |
| 胜利弹窗“领取”   | 进入 CTA 页面             |
| 失败弹窗“重新挑战” | 重置并重播 `lose` 流程       |
| 失败弹窗“退出挑战” | 进入 CTA 页面             |
| CTA 主按钮    | 调用可替换 clickthrough 方法 |

## 7.3 按钮限制

* 所有按钮必须防重复点击。
* 弹窗显示时，战斗区不得响应输入。
* CTA 显示后，时间轴必须停止。
* 重新挑战时不得保留旧定时器、旧特效、旧角色或旧事件监听。

## 8. 通用角色行为

角色状态：

```text
hidden
spawn
idle
move
attack
hit
die
exit
```

表现要求：

| 状态       | 视觉表现               |
| -------- | ------------------ |
| `spawn`  | 出现、缩放、闪光或游入        |
| `idle`   | 帧动画待机，或上下浮动        |
| `move`   | 朝目标移动，保留鱼类游动感      |
| `attack` | 攻击帧 + 前冲 / 水波 / 光效 |
| `hit`    | 闪白、抖动、短暂后退         |
| `die`    | 死亡帧、淡出、缩小或下沉       |
| `exit`   | 离开战斗区并销毁实例         |

每个角色必须支持安全中断：

```text
停止当前动画
→ 移除临时事件
→ 恢复或销毁显示对象
```

## 9. 时间轴事件模型

所有流程事件必须集中配置，不允许把大量 `setTimeout` 分散在 UI、角色或场景代码中。

推荐事件类型：

```text
spawnActor
spawnEnemyGroup
removeEnemyGroup
setActorState
actorAttack
actorHit
actorExit
showEffect
unlockEvolution
updateEvolutionProgress
showBoss
removeBoss
updateBossHp
showStamina
updateStamina
showTimer
updateTimer
showResult
showCta
playAudio
```

每个事件至少包含：

```text
at       触发时间，单位毫秒
type     事件类型
target   目标角色 / UI / 敌人
payload  额外参数
```

时间轴中的时间点为可调视觉目标，不应视为不可变业务规则。

# 10. 胜利流程：`mode=win`

## 10.1 目标流程

```text
初始骨架鱼形单位
→ 普通敌鱼战斗
→ 第 1 次进化
→ 第 2 次进化
→ 第 3 次进化
→ 第 4 次进化
→ 鱼王 Boss 出现
→ Boss 被击败
→ 奖励弹窗
→ CTA
```

## 10.2 初始 UI

顶部：

```text
左侧：关卡标题 / 第 185 关等样式文案
右侧：排行榜、通关记录样式入口
```

底部：

```text
标题：生物进化
进化卡：4 张
初始状态：均为 locked
每张卡包含：
- 单位头像
- 单位名称
- 解锁状态
- 经验进度条
```

战斗区：

```text
左侧：starter
右侧：普通敌鱼群
中部：海底背景与攻击空间
```

## 10.3 胜利时间轴

| 时间目标       | 事件                                |
| ---------- | --------------------------------- |
| 0.0s       | 初始化背景、HUD、进化卡、starter、普通鱼群        |

| 0.6–3.2s   | 重复攻击、命中、敌鱼退出、敌鱼补充                 |
| 2.8s       | 第 1 张进化卡经验接近满值                    |
| 3.3s       | 播放第 1 次进化特效                       |
| 3.5s       | evolution_1 出现并替代 starter         |
| 3.6s       | 第 1 张卡切换为 unlocked                |
| 3.8–5.1s   | evolution_1 持续攻击普通鱼群              |
| 5.2s       | 第 2 张卡经验完成，播放进化特效                 |
| 5.4s       | evolution_2 出现，第 2 张卡解锁           |
| 5.7–7.1s   | evolution_2 持续攻击普通鱼群              |

| 7.7–9.1s   | evolution_3 持续攻击普通鱼群              |

| 9.7s       | boss_win 从右侧出现                    |

| 14.2s      | Boss 血条归零                         |
| 14.3s      | Boss 播放死亡 / 退出特效                  |
| 14.5s      | 停止战斗，进入胜利结算                       |
| 15.0s      | 显示暗色遮罩与奖励弹窗                       |
| 15.3s      | 展示“恭喜获得”、奖励格与领取按钮                 |

## 10.4 Boss 战表现要求

Boss 战必须有明显层次：

```text
Boss 出场
→ 角色攻击
→ 攻击特效
→ Boss 命中特效
→ Boss 血条下降
→ 最后一击
→ Boss 死亡 / 消失
→ 胜利结算
```

不需要真实伤害计算。

血条变化必须由时间轴控制。

## 10.5 胜利奖励弹窗

弹窗元素：

```text
暗色全屏遮罩
标题：恭喜获得
角色插画或最终角色展示
多个奖励格
领取按钮
```

规则：

* 奖励内容可为固定展示，不需要真实背包。
* 点击“领取”后进入 CTA。
* 弹窗期间不得继续触发战斗事件。

## 10.6 胜利 CTA 页面

CTA 元素：

```text
顶部：你能吞噬多少鱼？
中部：final_actor 展示
底部：无需下载 点击即玩
主按钮：立即体验 / 点击即玩
```

CTA 行为：

```text
隐藏战斗层、HUD、进化卡和结算弹窗
展示 CTA 背景与主体角色
主按钮调用 clickthrough
无宿主跳转能力时，只提供按钮缩放与音效反馈
```

## 12. 重新挑战重置规则

点击“重新挑战”后，必须完整清理并恢复初始状态。

## 16. 验收标准

## 16.1 基础验收

* `?mode=win` 可正常启动。
* `?mode=lose` 可正常启动。
* 未传 mode 时默认进入 win。
* 720 × 1280 设计比例正确。
* 常见手机尺寸下关键 UI 不裁切。
* 原始资源未被修改。
* `Thumbs.db` 未参与资源加载。
* 项目可正常构建。

## 16.2 资源验收

* 主要角色至少可播放 idle、attack、die 或 ult。
* 至少一种攻击特效可播放。
* 至少一种命中特效可播放。
* 进化特效可通过现有资源或降级效果呈现。
* 角色动画帧顺序正确，无明显乱序跳帧。

## 16.3 胜利流程验收

* 四张进化卡按顺序解锁。
* 最终 Boss 出现。
* Boss 血条下降并归零。
* Boss 死亡或退出。
* 胜利奖励弹窗正常显示。
* 点击领取进入 CTA。
* CTA 页面不会继续播放战斗事件。

## 16.4 失败流程验收

* 四张进化卡按顺序解锁。
* 最终敌人进入主要对抗位置。
* 体力条和吞噬时间正常显示并下降。
* 最终角色失败后出现失败弹窗。
* 点击“重新挑战”可从 0 秒完整重播。
* 连续重试至少 10 次后，无重复角色、重复事件、重复倒计时或 UI 残留。
* 点击“退出挑战”进入 CTA。
