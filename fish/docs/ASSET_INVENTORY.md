# ASSET_INVENTORY.md

本文件记录 `assets/images` 目录下所有可用美术资源的完整清单。  
以文件扫描结果为准；`Thumbs.db` 和 `.DS_Store` 已排除。

---

## 1. 文件命名规范

### 角色动画帧（普通 fish）

```
fish_XXXXX/hero/fish_XXXXX-ACTION_NN.png
```

示例：
```
fish_11002/hero/fish_11002-idle_00.png
fish_11002/hero/fish_11002-idle_01.png
...
fish_11002/hero/fish_11002-atk_21.png
```

### 角色动画帧（skin_fish_151011 特殊）

`skin_fish_151011` **无 `hero/` 子目录**，帧文件直接放在顶层：

```
skin_fish_151011/fish151011-ACTION_NN.png
```

示例：
```
skin_fish_151011/fish151011-idle_00.png
skin_fish_151011/fish151011-ult_54.png
```

> **实现注意**：`gen-manifest.js` 脚本必须单独处理 `skin_fish_151011` 的扫描路径，不能假设所有角色都有 `hero/` 子目录。

### 特效动画帧

```
fish_XXXXX/eff_TYPE/eff_XXXXX_descriptor-start_NN.png
```

示例：
```
fish_13201/eff_atk/eff_13201_main_atk-start_00.png
fish_14104/eff_ult/eff_14104_main_ult-start_74.png
```

### UI 图片

```
assets/images/bg.png    战斗背景
assets/images/sk.png    次级背景/天空纹理
```

**帧序号**：0 起始，连续整数，无前导零对齐问题（但扫描时应自然排序）。

## 6. 角色槽位与资源映射建议

基于视频分析与资源特征，初步映射如下：

| 槽位 | 建议 Fish ID | 依据 |
|------|------------|------|
| `starter` | `fish_11002` | 帧数最多、有 ultb；外形简单适合初始角色 |
| `evolution_1` | `fish_11003` | eff_ult 帧数多（57帧），进化特效震撼 |
| `evolution_2` | `fish_15101` | idle/move 帧数较少，节奏更快 |
| `evolution_3` | `fish_15102` | 中等帧数，有 eff_atk + eff_hit |

| `enemy_lose` | `fish_13201` | 三种特效齐全，表现力最强；未出现在主角阵营列表中 |

> 以上仅为初始建议，最终映射以 `src/config/assetMapping.ts` 配置为准。  
> 若视觉测试后发现不匹配，直接修改 `assetMapping.ts` 无需改动业务代码。

## 7. 动作降级规则（资源缺失时）

| 目标动作 | 优先资源 | 缺失时降级方案 |
|--------|---------|-------------|
| `idle` | `hero/*-idle_*` | 首帧 + 轻微上下 Tween 浮动 |
| `move` | `hero/*-move_*` | 使用 `idle` + Tween 位移 |
| `attack` | `hero/*-atk_*` | 使用 `ult` 或 `idle` + 前冲 |
| `ult` | `hero/*-ult_*` | 使用 `atk` + 屏幕闪光强化 |
| `hit` | `eff_hit/*` | 角色白闪 + 抖动 + 短暂后退 |
| `die` | `hero/*-die_*` | 缩小 + 淡出 + 下沉 Tween |
| `eff_atk` | `eff_atk/*` | 简单水波 / 扩散光圈（程序生成）|
| `eff_ult` | `eff_ult/*` | 放大攻击特效 + 屏幕闪光 |
