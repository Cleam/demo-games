# ASSUMPTIONS.md

本文件记录 Phase 0 盘点阶段做出的所有假设性决策。  
凡无明确文档依据而需要做出合理判断的内容，均列于此，以便后续阶段发现偏差时快速定位和调整。

---

## A1. fish_13201 和 fish_13203 用于敌方角色

**背景**：`assets/images` 目录下存在 `fish_13201/` 和 `fish_13203/` 两个角色目录，但 GAME_FLOW.md §3.1 的"可用主角色资源池"中未列出这两个 ID。

**假设**：这两个角色不作为主角阵营成员，而是作为 `enemy_lose`（失败流程最终敌人）或 `boss_win`（胜利流程 Boss）的候选资源。

**理由**：
- `fish_13201` 拥有完整三种特效（eff_atk + eff_hit + eff_ult），表现力最强，适合最终 Boss 级别角色。
- `fish_13203` 也有 eff_atk + eff_ult，可作为备选。
- 两者均未在主流程角色列表中出现，合理推断为敌方或特殊用途。

**优先映射**：`enemy_lose` → `fish_13201`

**调整方式**：修改 `src/config/assetMapping.ts` 中 `enemy_lose` 字段即可，无需改动业务代码。

## A2. bg.png 用作战斗背景，居中裁切适配竖屏

**背景**：`bg.png` 尺寸为 3328×2048（横向 16:9），目标设计尺寸为 720×1280（竖屏 9:16）。

**假设**：bg.png 是战斗区的海底背景图，按竖屏居中裁切（取水平中部 720 宽度区域，垂直方向缩放至铺满 1280），不拉伸。

**实现方式**：Phaser 中通过 `setOrigin(0.5, 0.5)` + 计算 scale 因子，或通过 Phaser.GameObjects.Container + Camera.setBounds 控制裁切视窗。

---

## A3. sk.png 为次级背景层，待视觉测试后确认用途

**背景**：`sk.png` 尺寸为 2560×1440（横向），命名含义不明确（可能为 sky、skin、secondary background）。

**假设**：sk.png 是海底背景的次级装饰层（如水面光效、远景纹理），叠加在 bg.png 之上或之下。

**处理方式**：Stage 7 视觉打磨阶段加载并测试；若不符合视觉预期，可降级为纯色背景层或不使用。

## A4. skin_fish_151011 无特效资源，全部使用程序降级

**背景**：`skin_fish_151011/` 目录下无 `eff_atk/`、`eff_hit/`、`eff_ult/` 子目录。

**假设**：该皮肤变体的特效均采用 GAME_FLOW.md §3.4 动作降级规则处理：
- `eff_atk` 缺失 → 简单水波 / 程序圆圈扩散
- `eff_hit` 缺失 → 白闪 + Camera.shake(100ms)
- `eff_ult` 缺失 → 放大攻击特效 + 全屏白闪

若 `skin_fish_151011` 被分配为 `boss_win`，Boss 死亡特效同样使用降级方案。

## A5. 帧序号从 0 开始，连续整数，无缺帧

**背景**：实际扫描到的帧序列形如 `fish11002-idle_00.png`～`fish11002-idle_59.png`（共 60 帧）。

**假设**：所有帧序列均从 `_00` 开始，连续无间断，文件名末尾数字即为帧索引。

**处理方式**：`gen-manifest.js` 脚本按文件名自然排序输出帧列表；若发现缺帧，静默跳过并在控制台打出警告，不中断加载。

## A6. 无音频资源，音效接口静默实现

**背景**：`assets/` 目录下无任何音频文件（无 `.mp3`、`.ogg`、`.wav`）。

**假设**：当前版本无音效。

**实现方式**：预留 `AudioManager` 接口，所有 `playAudio` 类型的时间轴事件映射到空函数。当后续音频资源补充后，只需实现 `AudioManager` 内部逻辑，无需修改时间轴配置。

---

## A7. 无自定义字体资源，使用系统字体

**背景**：`assets/` 目录下无字体文件。

**假设**：UI 文字（关卡标题、进化卡名称、弹窗标题、CTA 文案）使用系统 sans-serif 字体。

**中文字体处理**：通过 Phaser `Text` 组件指定 `'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', sans-serif` 字体族，在常见手机上均可正常渲染中文。

**背景**：视频 1 显示 Boss 为"鱼王/刺豚"，外形与普通角色有明显差异。

## A9. 进化卡经验条为纯视觉表现，无真实计算

**背景**：GAME_FLOW.md §2.2 明确"不需要实现真实养成数值"。

**假设**：进化卡的经验进度条完全由时间轴驱动（`updateEvolutionProgress` 事件），进度值为配置中的固定序列（如 0% → 30% → 70% → 100%），不与战斗伤害计算挂钩。

---

## A10. 重试时使用 Phaser Scene restart，不手动逐一销毁

**背景**：GAME_FLOW.md §12 要求"重新挑战"后完整重置所有状态。

**假设**：最安全的实现方式是调用 `this.scene.restart()`（重启 GameScene），Phaser 会自动销毁所有 GameObject 并重新执行 create()。相比手动逐一 destroy，这样不容易遗漏。

**风险**：如果 GameScene 与 LoadingScene 之间有共享状态（如已加载的 texture），需确保 texture cache 在 restart 时不被清空，避免重新触发加载。具体实现时验证。

---
